### Code of Conduct
Please be respectful and professional in all interactions.

### How Can I Contribute?
- **Reporting Bugs**: Use GitHub Issues.
- **Suggesting Enhancements**: We welcome innovative ideas (especially in Auto-RAG and Multi-Modal domains).
### Pull Requests
We accept high-quality contributions via Pull Requests. To ensure your contribution meets **RJ Business Solutions** enterprise standards:
1.  **Issue Alignment**: Every PR must reference an existing open Issue.
2.  **Atomic Commits**: Use descriptive, atomic commits. Large "catch-all" commits will be rejected.
3.  **Documentation**: Update all relevant `.md` files in the root and `/research` directory if your changes affect the architectural logic.
4.  **Verification**: Include a `walkthrough.md` or similar evidence of testing for complex features.
5.  **Review**: All PRs require approval from at least one **RJ Business Solutions** core maintainer.

### Coding Standards
- Use type hints for all functions.
- Follow PEP 8 style guidelines.
- Add docstrings to all major classes and methods.
- Ensure any new retrieval strategies are performance-benchmarked.
