### Code of Conduct
Please be respectful and professional in all interactions.

### How Can I Contribute?
- **Reporting Bugs**: Use GitHub Issues.
- **Suggesting Enhancements**: We welcome innovative ideas (especially in Auto-RAG and Multi-Modal domains).
- **Pull Requests**:
  1. Fork the repo.
  2. Create a feature branch.
  3. Ensure your code passes all lint checks.
  4. Submit your PR for review by the RJ Business Solutions core team.

### Coding Standards
- Use type hints for all functions.
- Follow PEP 8 style guidelines.
- Add docstrings to all major classes and methods.
- Ensure any new retrieval strategies are performance-benchmarked.
